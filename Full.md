# Полная документация проекта MystVPN Bot

## Обзор проекта

**MystVPN Bot** - это полнофункциональный Telegram-бот, предоставляющий услуги VPN через удобный интерфейс. Проект представляет собой комплексное решение для предоставления безопасного и надежного доступа к интернету через VPN-соединения с поддержкой различных протоколов и способов оплаты.

### Название проекта
- **MystVPN**
- **Telegram-бот**: @MystVPN_bot
- **Цель проекта**: Предоставление безопасного и надежного доступа к интернету через Telegram-бота с поддержкой современных протоколов

### Основные функции
- Подписка на VPN-ключи с различными тарифными планами (от 1 месяца до 1 года)
- Поддержка множества языков (17 языков)
- Интеграция с платежными системами (YooKassa, Telegram Stars)
- Управление подписками пользователей
- Реферальная система с бонусами
- Подарочные коды
- Пробный период для новых пользователей
- Интеграция с XRay/3x-ui для управления VPN-подключениями
- Вебхук-сервер для обработки платежей
- Административная панель

## Архитектура проекта

### Общая архитектура
MystVPN Bot использует модульную архитектуру, состоящую из следующих основных компонентов:

1. **Telegram-бот** (основной интерфейс)
2. **Вебхук-сервер** (обработка платежей)
3. **Система хранения данных** (PostgreSQL, Redis)

5. **Службы приложения** (бизнес-логика)

## Конфигурация

### Файл .env
Проект использует переменные окружения для конфигурации. Ниже представлены все переменные:

#### Основные настройки бота:
- `BOT_TOKEN`: Токен Telegram-бота (обязательно)
- `BOT_USERNAME`: Имя пользователя бота (по умолчанию MystVPN_bot)

#### Настройки базы данных:
- `DATABASE_URL`: Строка подключения к PostgreSQL (например, postgresql+asyncpg://mystvpn:mystvpn_secret@localhost/mystvpn_bot)

#### Настройки Redis:
- `REDIS_HOST`: Хост Redis (по умолчанию localhost)
- `REDIS_PORT`: Порт Redis (по умолчанию 6379)

#### Настройки платежей:
- `YOOKASSA_ACCOUNT_ID`: ID аккаунта YooKassa
- `YOOKASSA_SECRET_KEY`: Секретный ключ YooKassa

#### Настройки вебхуков:
- `WEBHOOK_URL`: URL для получения вебхуков
- `WEBHOOK_SECRET`: Секретный ключ для проверки подписи вебхуков

#### Настройки прокси:
- `PROXY_HOST`: Хост SOCKS5 прокси
- `PROXY_PORT`: Порт SOCKS5 прокси
- `PROXY_LOGIN`: Логин прокси (если требуется)
- `PROXY_PASS`: Пароль прокси (если требуется)

#### Настройки администраторов:
- `ADMIN_IDS`: Список ID администраторов бота через запятую (например, 123456789,987654321)

#### Настройки XRay/VPN:
- `XRAY_HOST`: IP-адрес или домен сервера XRay
- `XRAY_PORT`: Порт панели XRay (по умолчанию 54321)
- `XRAY_USERNAME`: Имя пользователя XRay панели
- `XRAY_PASSWORD`: Пароль XRay панели
- `XRAY_INBOUND_ID`: ID инбаунда VLESS для создания клиентов
- `XRAY_XHTTP_INBOUND_ID`: ID инбаунда XHTTP для резервного протокола
- `XRAY_ADDRESS`: Альтернативный адрес XRay (если отличается от XRAY_HOST)

#### Настройки Reality:
- `REALITY_PRIVATE_KEY`: Приватный ключ для пересоздания Reality-инбаундов
- `REALITY_PUBLIC_KEY`: Публичный ключ для пересоздания Reality-инбаундов
- `REALITY_SHORT_ID`: Short ID для Reality-протокола
- `REALITY_DEST`: Назначение для Reality (по умолчанию vk.com:443)
- `REALITY_SNI`: SNI для Reality (по умолчанию vk.com)

#### Прочие настройки:
- `WEBHOOK_PORT`: Порт вебхук-сервера (по умолчанию 8090)
- `SUB_DOMAIN`: Домен для подписок (если используется отдельный домен)
- `SUB_PATH`: Путь к endpoint подписки (по умолчанию /sub/)

## База данных и модели

### ORM
Проект использует SQLAlchemy 2.0 с асинхронными сессиями для работы с базой данных.

### Модели данных

#### User модель
- `id`: ID пользователя в Telegram (первичный ключ)
- `username`: Имя пользователя
- `first_name`: Имя
- `language`: Язык интерфейса
- `is_admin`: Флаг администратора
- `is_banned`: Флаг блокировки
- `referral_id`: ID пользователя, пригласившего
- `balance`: Баланс пользователя (в рублях)
- `created_at`: Дата регистрации
- `updated_at`: Дата последнего обновления

#### Subscription модель
- `id`: Уникальный ID подписки
- `user_id`: ID пользователя
- `plan`: Ключ тарифного плана
- `status`: Статус (pending, active, expired)
- `start_date`: Дата начала
- `end_date`: Дата окончания
- `vpn_key`: VPN-ключ (VLESS/VMess/Trojan URL)
- `sub_url`: URL подписки
- `trial_used`: Флаг использования пробного периода
- `created_at`: Дата создания

#### Payment модель
- `id`: Уникальный ID платежа
- `user_id`: ID пользователя
- `subscription_id`: ID связанной подписки
- `plan`: Ключ тарифного плана
- `amount`: Сумма платежа
- `method`: Метод оплаты
- `external_id`: Внешний ID платежа (например, от YooKassa)
- `status`: Статус (pending, completed, failed)
- `created_at`: Дата создания

#### GiftCode модель
- `id`: Уникальный ID подарочного кода
- `code`: Сам код
- `plan_key`: Ключ тарифного плана
- `buyer_id`: ID пользователя, создавшего код
- `recipient_id`: ID пользователя, использовавшего код
- `is_paid`: Флаг оплаты
- `is_used`: Флаг использования
- `created_at`: Дата создания
- `used_at`: Дата использования

#### PromoCode модель
- `id`: Уникальный ID промо-кода
- `code`: Сам код
- `discount_percent`: Процент скидки
- `usage_limit`: Лимит использований
- `usage_count`: Количество использований
- `is_active`: Флаг активности
- `expires_at`: Дата окончания действия
- `created_at`: Дата создания

#### Donation модель
- `id`: Уникальный ID доната
- `user_id`: ID пользователя
- `username`: Имя пользователя
- `first_name`: Имя пользователя
- `amount`: Сумма доната
- `created_at`: Дата создания

## Сервисы

### Основные сервисы

#### UserService
- `get_or_create`: Получение или создание пользователя
- `get`: Получение пользователя по ID
- `update_language`: Обновление языка интерфейса
- `ban/unban`: Блокировка/разблокировка пользователя
- `make_admin`: Назначение администратора

#### SubscriptionService
- `create_pending`: Создание ожидающей подписки
- `activate`: Активация подписки
- `is_trial_available`: Проверка доступности пробного периода
- `save_key`: Сохранение VPN-ключа и URL подписки
- `create_trial`: Создание пробной подписки
- `get_current`: Получение текущей активной подписки
- `notify_expiry`: Уведомление об окончании подписки

#### PaymentService
- `create_yookassa_payment`: Создание платежа через YooKassa
- `get_by_ext_id`: Получение платежа по внешнему ID
- `complete`: Завершение платежа
- `check_yookassa`: Проверка статуса платежа через YooKassa API
- `get_user_payments`: Получение истории платежей пользователя

#### XrayService
- `create_client`: Создание VPN-клиента в XRay
- `remove_client`: Удаление VPN-клиента из XRay
- `reset_client`: Сброс и пересоздание VPN-ключа
- `_build_key`: Генерация VPN-ключа (VLESS/VMess/Trojan URL)
- `test_connection`: Проверка подключения к XRay
- `recreate_inbound`: Пересоздание инбаунда Reality (при сбое)
- `_extract_uuid`: Извлечение UUID из существующего ключа

#### NotificationService
- `run_notification_loop`: Запуск цикла уведомлений
- `send_expiry_notifications`: Отправка уведомлений об окончании подписки
- `send_expiry_discount_promo`: Отправка промо-скидки в день окончания

#### ReferralService
- `process_referral`: Обработка реферала при регистрации
- `get_referral_stats`: Получение статистики по рефералам
- `award_referral_bonus`: Начисление бонуса за реферала

#### PromoService
- `get_by_code`: Получение промо-кода по коду
- `apply_promo_code`: Применение промо-кода
- `save_discount`: Сохранение информации о скидке

#### GiftService
- `create`: Создание подарочного кода
- `get_by_code`: Получение подарочного кода по коду
- `activate`: Активация подарочного кода
- `get_by_payment_ext_id`: Получение кода по внешнему ID платежа

#### DonationService
- `record_rub`: Запись рублевого доната
- `get_total_donations`: Получение общей суммы донатов

### Особенности XRayService
- Интеграция с 3x-ui/XRay панелью
- Поддержка VLESS/Reality протокола (устойчив к блокировкам)
- Поддержка резервного протокола XHTTP
- Генерация уникальных ключей для каждого пользователя
- Поддержка подписок через URL
- Автоматическое восстановление соединения
- Механизм "watchdog" для проверки доступности инбаунда
- Возможность автоматического пересоздания инбаунда с российскими настройками

### Структура обработчиков
Каждый обработчик реализован в виде отдельного модуля с собственным маршрутизатором (Router) из Aiogram 3.x. Все обработчики импортируются в главный файл [main.py](main.py) и регистрируются в диспетчере.

Пример структуры обработчика:
```python
from aiogram import Router, F
from aiogram.filters import Command
from aiogram.types import Message, CallbackQuery

router = Router()

@router.message(Command("start"))
async def cmd_start(message: Message):
    # Логика обработки команды /start
    pass

@router.callback_query(F.data == "some_callback")
async def callback_handler(callback: CallbackQuery):
    # Логика обработки колбэка
    pass
```

## Функциональные возможности

### Пробный период
- Новые пользователи получают 3 дня бесплатного VPN
- Возможность продлить подписку после пробного периода
- Автоматическое предложение пробного периода при первом запуске
- Ограничение одного пробного периода на пользователя

### Подписки
- Различные тарифные планы (от 1 месяца до 1 года)
- Автоматическое уведомление о скором окончании подписки (за 5, 1 и 0 дней)
- Возможность продления по скидке в день окончания (через промокод)
- Поддержка пробных подписок
- Отображение оставшихся дней подписки
- Возможность сброса VPN-ключа

### Платежи
- Интеграция с YooKassa
- Поддержка оплаты в Telegram Stars
- Автоматическое подтверждение платежей через вебхуки
- Проверка подлинности вебхуков через HMAC-подпись
- История платежей для каждого пользователя
- Отмена неоплаченных платежей

### Реферальная система
- Пользователь получает 7 дней за каждого приглашенного друга
- При достижении 10 рефералов - дополнительные 30 дней
- Автоматическое начисление бонусов
- Отслеживание реферальной статистики
- Поддержка deeplink-рефералов

### Подарочные и промо-коды
- Генерация и активация подарочных кодов
- Применение промо-кодов со скидкой
- Отслеживание использования кодов
- Подарочные коды как отдельный тип платежа
- Возможность делиться ссылками на подарочные коды

### Администрирование
- Панель администратора с расширенными возможностями
- Управление пользователями (бан, разбан, назначение прав)
- Создание подарочных и промо-кодов
- Массовая рассылка уведомлений
- Статистика использования и платежей
- Возможность сброса ключей пользователям
- Управление реферальной программой

docker-compose up -d --build

# Проверка состояния
docker-compose ps

# Просмотр логов
docker-compose logs -f

## Аналитика и статистика

Для получения подробной аналитики и статистики о работе бота, вы можете использовать следующие методы и SQL-запросы:

### 1. База пользователей и подписок

#### Количество пользователей:
```sql
-- Всего зарегистрированных пользователей
SELECT COUNT(*) AS total_users FROM users;

-- Активные пользователи за последние 30 дней
SELECT COUNT(DISTINCT u.id) AS active_users_last_30_days
FROM users u
JOIN subscriptions s ON u.id = s.user_id
WHERE s.updated_at >= NOW() - INTERVAL '30 days'
AND s.status = 'active';
```

#### Активные подписки:
```sql
-- Активные подписки по тарифам
SELECT 
    plan,
    COUNT(*) AS active_count
FROM subscriptions 
WHERE status = 'active' 
    AND end_date > NOW()
GROUP BY plan;
```

#### Пробные подписки:
```sql
-- Пробные активировано за последнюю неделю
SELECT COUNT(*) AS trials_last_week
FROM subscriptions
WHERE trial_used = true
    AND created_at >= NOW() - INTERVAL '7 days';

-- Конверсия из пробных в платные
WITH trials AS (
    SELECT user_id
    FROM subscriptions
    WHERE trial_used = true
        AND created_at >= NOW() - INTERVAL '7 days'
),
converted AS (
    SELECT DISTINCT t.user_id
    FROM trials t
    JOIN subscriptions s ON t.user_id = s.user_id
    WHERE s.plan IS NOT NULL
        AND s.status = 'active'
        AND s.trial_used = false
)
SELECT 
    COUNT(trials.user_id) AS total_trials,
    COUNT(converted.user_id) AS converted_trials,
    ROUND(COUNT(converted.user_id) * 100.0 / COUNT(trials.user_id), 2) AS conversion_rate
FROM trials
LEFT JOIN converted ON trials.user_id = converted.user_id;
```

### 2. Платежи

#### Доход за апрель:
```sql
-- Общий доход за апрель 2026
SELECT SUM(amount) AS total_revenue_april
FROM payments
WHERE status = 'completed'
    AND EXTRACT(MONTH FROM created_at) = 4
    AND EXTRACT(YEAR FROM created_at) = 2026;

-- Доход по дням за апрель
SELECT 
    DATE(created_at) AS day,
    SUM(amount) AS daily_revenue
FROM payments
WHERE status = 'completed'
    AND EXTRACT(MONTH FROM created_at) = 4
    AND EXTRACT(YEAR FROM created_at) = 2026
GROUP BY DATE(created_at)
ORDER BY day;

-- Доход по неделям за апрель
SELECT 
    DATE_TRUNC('week', created_at) AS week,
    SUM(amount) AS weekly_revenue
FROM payments
WHERE status = 'completed'
    AND EXTRACT(MONTH FROM created_at) = 4
    AND EXTRACT(YEAR FROM created_at) = 2026
GROUP BY DATE_TRUNC('week', created_at)
ORDER BY week;
```

#### Средний чек:
```sql
-- Средний чек
SELECT AVG(amount) AS avg_check
FROM payments
WHERE status = 'completed';
```

#### Методы оплаты:
```sql
-- Доля методов оплаты
SELECT 
    method,
    COUNT(*) AS count,
    COUNT(*) * 100.0 / (SELECT COUNT(*) FROM payments WHERE status = 'completed') AS percentage
FROM payments
WHERE status = 'completed'
GROUP BY method;
```

### 3. Трафик и воронка

#### Уникальные пользователи, запускающие бота:
```sql
-- Уникальные запуски в день
SELECT 
    DATE(created_at) AS day,
    COUNT(DISTINCT user_id) AS unique_starts
FROM (
    -- Предполагаем, что логи запусков хранятся где-то
    -- В реальности нужно будет добавить таблицу событий
    SELECT user_id, created_at FROM users -- заглушка
) AS start_events
WHERE created_at >= NOW() - INTERVAL '30 days'
GROUP BY DATE(created_at)
ORDER BY day;
```

#### Воронка пользователей:
```sql
-- Пользователи, дошедшие до выбора тарифа
-- (предполагается наличие соответствующего логирования событий)
SELECT COUNT(DISTINCT user_id) AS reached_tariff_selection
FROM events -- заглушка для таблицы событий
WHERE event_type = 'tariff_selected'
    AND created_at >= NOW() - INTERVAL '30 days';

-- Брошенные корзины
SELECT COUNT(DISTINCT user_id) AS abandoned_checkout
FROM events -- заглушка для таблицы событий
WHERE event_type = 'checkout_initiated'
    AND NOT EXISTS (
        SELECT 1 FROM payments p 
        WHERE p.user_id = events.user_id 
            AND p.created_at > events.created_at
            AND p.status = 'completed'
    );
```

#### Источники трафика:
```sql
-- Если в реферальных ссылках хранится источник
SELECT 
    source,
    COUNT(*) AS user_count
FROM users
WHERE source IS NOT NULL
GROUP BY source;
```

### Альтернативные способы получения статистики

Помимо SQL-запросов, статистику можно получить:

1. **Через админ-панель бота** (если реализована):
   - Запустите бота с правами администратора
   - Перейдите в раздел статистики

2. **Через API YooKassa**:
   ```bash
   curl -X GET \
     -u "{account_id}:{secret_key}" \
     "https://api.yookassa.ru/v3/payments?created_at.gte=2026-04-01&created_at.lt=2026-05-01&status=captured"
   ```

3. **Через панель 3x-ui**:
   - Веб-интерфейс панели содержит информацию о трафике пользователей
   - Доступен экспорт данных по использованию

4. **Через Redis** (если хранятся сессии и события):
   - Используйте `redis-cli` для получения данных
   - Проверьте ключи, начинающиеся с `session:` или `stats:`

5. **Через лог-файлы**:
   - Анализ файла `bot.log` может содержать информацию о событиях
   - Используйте `grep`, `awk` и другие инструменты для фильтрации

Эти запросы и методы помогут вам получить полную картину использования бота, доходов и эффективности маркетинговых каналов.
