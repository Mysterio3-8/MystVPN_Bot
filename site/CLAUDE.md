# CLAUDE.md — keybest.cc (лендинг и гайд)

## Что это
Статический сайт keybest.cc — лендинг + страницы с инструкциями для юзеров.

## Структура
```
site/
 ├─ index.html       ← главная (лендинг, оффер, кнопка «Открыть бота»)
 ├─ pages/           ← гайды по подключению (iOS, Android, Windows, Mac)
 ├─ css/             ← стили (вынесены из inline)
 ├─ js/              ← скрипты
 └─ ARCHITECTURE.md  ← как устроен сайт
```

## Деплой
Сайт раздаётся nginx-ом из контейнера `nginx` (`docker-compose.yml`).
В deploy.yml автоматически копируется в `/var/www/guide/` для отдельного префикса.

После пуша в master → GitHub Actions → автообновление сайта на сервере.

## Маршруты
- `https://keybest.cc/` → `index.html` (лендинг)
- `https://keybest.cc/guide/` → `/var/www/guide/` (старый путь, может уйти)
- `https://keybest.cc/pages/<slug>` → конкретный гайд

## OPSEC правила (КРИТИЧНО)
**Запрещено в публичном тексте сайта:**
- Слова: VLESS, Reality, xray, прокси, протокол, обфускация, SNI, fallback
- Технические детали как именно работает обход
- Скриншоты с реальными адресами серверов / портами

**Можно и нужно:**
- «Обход блокировок», «доступ ко всему интернету»
- «Готовы к Чебурнету», «работает когда другие падают»
- «Высокая скорость», «без логов», «гарантия возврата»
- Социальные пруфы, отзывы, кейсы

## Контент стратегия
1. **Главная** — оффер в первом экране, цена, кнопка в бот
2. **Гайды** — по одному на платформу, простой язык, без технотерминов
3. **FAQ** — закрывает возражения («законно?», «безопасно?», «что если не подойдёт?»)
4. **Блог** (будущее) — посты про доступ к сервисам (Notion, ChatGPT, Spotify) для SEO

## Команды
```bash
# Локально — открыть в браузере
start index.html   # Windows
open index.html    # Mac

# На сервере: при пуше в master автодеплой через GitHub Actions
git push origin master
```

## Когда менять
- Изменение тарифов → обновить index.html (секция цен)
- Новые отзывы → обновить блок социального пруфа
- Новые гайды → добавить в `pages/`
