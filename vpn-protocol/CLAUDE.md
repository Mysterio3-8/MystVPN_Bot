# CLAUDE.md — VPN Protocol (улучшения transport-слоя)

## Статус
🟡 **Не начато.** Папка-заготовка для собственных протокольных улучшений.

## Зачем
Сейчас используется стандартный xray-core (внешняя зависимость). Когда бизнес поднимется, может понадобиться:
- Кастомные обфускации поверх стандартного транспорта
- A/B тестирование разных fallback-цепочек
- Свои метрики латентности / packet loss
- Адаптивный выбор транспорта на стороне клиента

## Возможные направления (черновик)
1. **Транспорт-абстракция** — единый интерфейс над xray / wg / future
2. **Probe-сервис** — клиент перед коннектом «нюхает» что не залочено и выбирает оптимум
3. **Health-rotation** — серверная сторона быстро переключает inbound при сигнатурном детекте

## OPSEC
- Этот модуль = **самое чувствительное** в проекте
- Код в этой папке — приватный, никаких public-репозиториев
- Никакой документации в публичных местах (README в Github = только заглушка)
- Конкретные SNI / порты / fallback цепи — ТОЛЬКО в .env / vault, не в коде

## Связи
- `bot/services/xray_service.py` — текущая реализация управления через 3x-ui API
- `vpn-app/vpn-engine/` — клиентская часть, использует библиотеки отсюда

## Когда начинать
**Когда у нас 500+ платящих и появится первый явный сигнал детекта.** Раньше — преждевременная оптимизация.
