## 🔐 1. VPN-стек — что работает в России (апрель 2026)

Ситуация в РФ резко изменилась. ТСПУ активно блокирует стандартные протоколы.

### ✅ Работает — используй это:

| Протокол | Статус | Клиент |
|---|---|---|
| **VLESS + XHTTP** | ✅ Работает | v2rayTUN, Hiddify |
| **Hysteria2** | ✅ Работает (UDP) | v2rayTUN, Hiddify, NekoBox |
| **AmneziaWG 2.0** | ✅ Работает | Amnezia клиент |
| **VLESS + Reality** | ⚠️ Частично (только UDP или редкий SNI) | v2rayTUN |
| **Shadowsocks-2022** | ✅ Работает при правильной настройке | Hiddify |

### ❌ Заблокировано — не использовать:

| Протокол | Статус |
|---|---|
| WireGuard стандартный | ❌ Заблокирован |
| OpenVPN | ❌ Заблокирован |
| VLESS + Reality по TCP | ❌ Заблокирован с 17 февраля 2026 |
| AmneziaWG (старая версия) | ❌ Заблокирован |

### 🎯 Рекомендуемый стек для запуска:

```
Сервер: 3x-ui (панель управления, бесплатно)
Основной протокол: VLESS + XHTTP (маскировка под HTTPS)
Резервный: Hysteria2 (UDP, обходит TCP-фильтры)
Клиент пользователя: v2rayTUN / Hiddify (ссылка-подписка)
```

**Ключевое преимущество:** Пользователь получает одну ссылку-подписку. При блокировке одного протокола, клиент автоматически переключается на резервный. Это твоё конкурентное преимущество — "ВПН который всегда работает".

## 🖥 2. Инфраструктура на Hetzner — максимум за $10

### Почему Hetzner — единственный правильный выбор:
- Лучшее соотношение цена/качество в Европе (DigitalOcean и Vultr дороже в 1.5–2×)
- Helsinki = минимальный пинг для РФ (~20–30ms vs ~80ms у Frankfurt)
- **20 TB трафика в месяц включено** — конкуренты дают 1–2 TB
- Не блокируется РКН (в отличие от части AWS/GCP IP)
- Живое ресайзирование серверов (можно проапгрейдить без переезда)

### ✅ Стартовая конфигурация — CX32, не CX22!

```
Сервер: Hetzner CX32  ← берём это, не CX22
├── ЦПУ: 4 vCPU (Intel)       ← в 2× больше ядер
├── RAM: 8 GB                  ← в 2× больше памяти
├── Диск: 80 GB SSD            ← в 2× больше диска
├── Трафик: 20 TB/месяц        ← одинаково с CX22
├── Локация: Helsinki (Финляндия)
└── Цена: €8.49/мес ≈ $9.20/мес
```

**Почему CX32, а не дешевле CX22 за €3.79:**
- CX22 упрётся в потолок при 80–100 одновременных VPN-соединениях
- CX32 комфортно держит 300–400 одновременных пользователей
- Разница €4.70 ($5) — смешная при доходе даже $100/мес
- Не нужно будет делать срочный переезд в первый месяц

**На одном CX32 разворачиваем:**
1. Telegram-бот (aiogram + PostgreSQL + Redis)
2. 3x-ui панель (VLESS + XHTTP + Hysteria2)
3. Nginx (обратный прокси, маскировка под HTTPS)

### Масштабирование по мере роста (реинвестиция из прибыли):

> 💡 **Маржа при 3000 юзерах: ~$6930 из $7000 дохода = 99%** — расходы на хостинг ничтожны

### 🌍 Локации для масштабирования (в порядке приоритета для РФ):

## 🗓 7. Пошаговый план деплоя на Hetzner

### Шаг 1: Создать аккаунт и сервер
```bash
# Зарегистрироваться на hetzner.com (нужна карта Visa/Mastercard)
# Создать проект → Add Server
# Выбрать: CX22 | Ubuntu 22.04 | Helsinki | SSH Key
# Получить IP-адрес сервера
```

### Шаг 2: Установить 3x-ui панель
```bash
ssh root@ВАШ_IP
bash <(curl -Ls https://raw.githubusercontent.com/MHSanaei/3x-ui/master/install.sh)
# Следовать инструкциям установщика
# Запомнить: порт панели, логин, пароль
```

### Шаг 3: Настроить VLESS + XHTTP
```
В 3x-ui:
1. Inbounds → Add Inbound
2. Protocol: VLESS
3. Transport: XHTTP
4. Security: None (маскируем через Nginx)
5. Включить Subscription URL
```

### Шаг 4: Настроить Nginx (маскировка)
```nginx
# Весь трафик выглядит как обычный HTTPS-сайт
server {
    listen 443 ssl;
    server_name ВАШ_ДОМЕН;
    # Реальный трафик идёт в 3x-ui, снаружи — обычный сайт
}
```

### Шаг 5: Задеплоить Telegram-бота
```bash
# Клонируй репозиторий на сервер
git clone ВАШ_РЕПО /opt/mystvpn-bot
cd /opt/mystvpn-bot

# Настрой .env
cp .env.example .env
nano .env  # Заполни BOT_TOKEN, DB_URL, REDIS_URL

# Запусти через Docker или systemd
python main.py
```

### Шаг 6: Интеграция бота с 3x-ui
```python
# В боте используй 3x-ui API для:
# - Создания нового ключа при оплате
# - Деактивации ключа при истечении
# - Получения статистики использования
# API доступен на: http://localhost:ПОРТ/panel/api/inbounds/...
```
