**Нужно переключиться на один из:**
| Протокол | Статус | Клиент |
|---|---|---|

| **AmneziaWG 2.0** | ✅ Работает | Amnezia клиент |
| **VLESS + Reality + UDP** | ⚠️ Частично | v2rayTUN |

**Рекомендуемый стек:**
```
Основной: VLESS + XHTTP (маскировка под HTTPS через Nginx на 443)
Резервный: Hysteria2 (UDP, обходит TCP-фильтры)
Клиент: v2rayTUN / Hiddify (по subscription link)
```
Пользователь получает одну subscription ссылку — при блокировке одного протокола клиент автопереключается.

- Убедиться что `sub_url` **всегда** выдаётся при создании ключа
- В 3x-ui: Fingerprint поменять с `qq` на `chrome`

## 🔄 Ротация ключей (когда меняешь сервер/IP)

**При subscription link** — ротация минимальна. Но если меняешь IP:

### Алгоритм grace period (24 часа):
1. Admin запускает команду `/admin_rotate_keys`
2. Для каждого активного пользователя:
   - Создаётся новый ключ (записывается в `new_vpn_key`, `new_sub_url`)
   - Устанавливается `key_rotation_deadline = now + 24h`
3. Пользователи получают пуш: _"Обновили инфраструктуру. Зайди в кабинет — новый ключ уже ждёт. Старый перестанет работать через 24ч."_
4. В кабинете показываются **оба ключа**: старый + новый с баннером
5. Cron через 24ч: удаляет старых клиентов из 3x-ui, переносит новый в основной
6. Второй пуш: _"Старый ключ отключён. Если не переключился — нажми /cabinet"_

**Нужно добавить в модель `Subscription`:**
```python
new_vpn_key: str | None  # новый ключ при ротации
new_sub_url: str | None
key_rotation_deadline: datetime | None
```
